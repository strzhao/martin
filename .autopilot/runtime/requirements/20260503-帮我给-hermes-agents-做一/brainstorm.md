# 大众点评 Skill — Deep Design Q&A 记录

## 需求摘要

为 Hermes Agent 制作一个 Skill，基于图片和音频生成高质量大众点评内容，内置评分体系，低分自动迭代优化。

## Q&A 结果

### Q1: 输入方式
**选择**: 本地文件路径
- 图片放在本地目录，通过 `vision_analyze` 工具分析
- 音频文件通过 whisper 技能转写
- Skill 自动判断输入类型

### Q2: 输出格式
**选择**: 纯文本 + 评分
- 自然语言评论（符合大众点评风格）
- 附带 1-5 星综合评分
- 四维度分项评分（具体性、生动性、实用性、真实性）

### Q3: 评分维度
**选择**: 综合型（四维度）
| 维度 | 说明 | 分值 |
|------|------|------|
| 具体性 | 是否提到具体菜品、价格、位置 | 1-5 |
| 生动性 | 描述是否让人垂涎、有画面感 | 1-5 |
| 实用性 | 对其他食客是否有参考价值 | 1-5 |
| 真实性 | 读起来是否像真人写的而非 AI | 1-5 |
| **总分** | | **0-20** |

- 阈值：总分 < 14 触发重新优化
- 阈值：任何单项 < 3 触发该维度专项修正

### Q4: 迭代优化策略
**选择**: 逐维度修正
- 定位低分维度 → 针对性修改 → 重新评分
- 最多 3 轮迭代
- 仍未达标时标记问题维度，交由用户判断

### Q5: 运营范围
**选择**: 纯内容生成
- 不涉及浏览器自动化发帖
- 用户自行复制粘贴到大众点评

### Q6: 技术架构
**选择**: 方案 B — Agent + 独立评审子代理

```
用户提供图片+音频 → vision_analyze + whisper →
初稿生成 → delegate子代理评分(独立上下文) →
逐维度诊断 → 主Agent修正 →
二次评审(同评审代理) →
输出: 评论文本 + 评审报告
```

## 关键技术决策

1. **视觉能力依赖**: 用户当前使用 DeepSeek provider 不支持原生视觉，需要经由 `vision_analyze` 工具（使用 Gemini Flash）处理图片
2. **音频处理**: 依赖 whisper 技能或系统转录工具
3. **子代理隔离**: 使用 Hermes 的 `delegate_task` 工具实现独立评审
4. **迭代上限**: 3 轮硬限制，防止死循环
5. **Skill 位置**: `~/.hermes/skills/productivity/dianping-review/`
