# 交叉验证规则

## 三维评分对照

每个餐厅/景点从以下维度评分：

| 维度 | 数据源 | 权重 |
|------|--------|------|
| 客观评分 | 高德 API POI 评分 + 高德状元榜排名 | 高 |
| 平台口碑 | dianping shop 总分+口味+环境+服务 | 高 |
| 用户热度 | xhs search 点赞数 + bilibili video 播放量 | 中 |
| 真实评价 | bilibili subtitle 字幕内容分析 | 中（口碑锚定） |

## 置信度判定

### 🟢 高置信度（优先推荐）
- 同一餐厅被 **≥ 2 个独立源** 认可
- 且 dianping 评分 ≥ 4.0
- 且无 bilibili 字幕中的明显差评

### 🟡 中置信度（可推荐，标注风险）
- 仅 1 个源提到，但有完整评分数据
- 或 ≥ 2 个源提到但评分 < 4.0

### 🔴 低置信度（不推荐）
- 仅 1 个源，无评分数据
- bilibili 字幕中明确踩雷

## 冲突处理

当不同源评价矛盾时：

| 场景 | 处理 |
|------|------|
| 小红书好评 + 点评低分 | 以点评评分为准（点评评分更难刷） |
| 点评高分 + B站字幕差评 | 在聚合卡片中同时展示两面评价 |
| 高德评分低 + 多源好评 | 高德评分可能滞后，数据不足时采信多源 |

## 信号强度计算

```
信号强度 = (dianping评分 × 0.4) + (高德状元榜排名倒数 × 0.2) + (xhs点赞数归一化 × 0.2) + (bilibili播放量归一化 × 0.2)
```

不强制计算，作为排序参考即可。
