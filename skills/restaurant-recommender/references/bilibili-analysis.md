# B站字幕分析策略

## 数据获取

```bash
opencli bilibili search "<目的地> <美食/探店>" --window background
opencli bilibili subtitle "<bvid>" --window background
```

## 字幕提取流程

1. **搜索**获取视频列表（含 BV号、播放量、UP主、标题）
2. **筛选**：优先播放量 > 50K 且 UP 主是专业厨子/探店博主
3. **字幕**获取逐句字幕（含时间戳）
4. **提取**关键信息

## 关键信息提取模板

从字幕文本中识别：

| 信息类型 | 识别模式 | 示例 |
|----------|----------|------|
| 餐厅名 | "来吃...", "到了...", "这家..." | "前面那块就是老胡子" |
| 菜品名 | "点了一个...", "来一个..." | "来一个白切鹅", "来一个猪蹄" |
| 价格 | "¥<数字>", "<数字>块", "<数字>元" | "40", "180", "一共35" |
| 好评 | "好吃", "不错", "绝了", "爱了" | "全程无槽点，样样都好吃" |
| 差评 | "不推荐", "太贵", "不好吃", "踩雷" | "18块钱就算了啊", "真吃不完" |
| 客观描述 | 口感/味道形容词 | "软烂脱骨", "脆嫩鲜甜", "酱卤香味很冲" |

## 噪声过滤

字幕中包含：
- **BGM 歌词**（分段标记 `♪ ... ♪`）→ 忽略
- **闲聊/过渡**（"这太热了", "继续逛"）→ 忽略
- **重复/口吃**（"很很", "你看看那边"）→ 忽略

## 优先级 UP 主

优先采信以下类型的 UP 主评价：
- 专业厨子探店（如实评价，会指出技术细节）
- 本地人日常 vlog（真实消费，非商业合作）
- 播放量 > 100K 的大 UP（评价更客观）

商业合作视频（标题含"广告"/"合作"）直接排除。
