# 五维度评分标准 v4.0

本文档供独立评审子代理使用。评审时逐维度打分，每项 1-5 分，总分 25 分。

## 风格参照

评审时以「老高+隋卞·温和专业型」为标杆：
- 老高式：温和态度、知识自然融入、批评不刺耳
- 隋卞式：技术眼光、从做法拆解、不讲空话
- 核心：专业做底 + 亲切做壳

## 评分维度详解

### 维度 1: 具体性 (Specificity)

衡量评论是否包含具体、可验证的细节。

| 分数 | 标准 | 示例 |
|------|------|------|
| 1 分 | 无任何具体信息 | "菜很好吃，环境不错" |
| 2 分 | 提到了菜系类型，但无具体菜名 | "粤菜做得不错" |
| 3 分 | 列出了菜名，但缺价格或做法细节 | "点了水煮鱼和宫保鸡丁" |
| 4 分 | 菜名+价格/分量，但细节不全 | "水煮鱼68元，分量足，麻辣味够" |
| 5 分 | 菜名+价格+做法+食材来源+分量全部具体 | "水煮鱼68元，用新鲜草鱼现杀，一斤半左右，麻辣鲜香层次分明" |

检查要点：
- [ ] 每道推荐菜是否标注了价格？
- [ ] 是否描述了菜品的做法或烹饪方式？
- [ ] 是否提到了食材的具体特征（新鲜度、品种、来源）？
- [ ] 分量描述是否具体（而非"分量足"这种模糊表述）？

### 维度 2: 生动性 (Vividness)

衡量评论是否让读者产生画面感和食欲。

| 分数 | 标准 | 示例 |
|------|------|------|
| 1 分 | 纯陈述，毫无画面感 | "味道还可以" |
| 2 分 | 有1-2个形容词 | "很好吃，很香" |
| 3 分 | 有感官描述但比较表面 | "鱼肉嫩滑，汤汁浓郁" |
| 4 分 | 多感官描述（视觉+味觉/嗅觉） | "端上桌时还在咕嘟冒泡，红油翻滚，花椒的麻香扑面而来" |
| 5 分 | 五感俱全，读之垂涎，有独特的比喻或表达 | "鱼肉像豆腐一样滑进嘴里，麻辣的后劲像小锤子一下下敲在舌头上" |

检查要点：
- [ ] 是否使用了视觉描述（颜色、形态、摆盘）？
- [ ] 是否使用了嗅觉/味觉描述（香气、口感、层次）？
- [ ] 是否有画面感的动词（咕嘟、翻滚、爆汁、化开）？
- [ ] 表达是否独特（而非"入口即化"等陈词滥调）？

### 维度 3: 实用性 (Usefulness)

衡量评论对其他食客的参考价值。

| 分数 | 标准 | 示例 |
|------|------|------|
| 1 分 | 纯个人感受，无参考价值 | "我爱死这家店了" |
| 2 分 | 有基本信息但无决策帮助 | "人均100左右" |
| 3 分 | 有价格+位置+推荐菜 | "在龙湖天街3楼，人均120，推荐杭甬一鲜汤" |
| 4 分 | 有避坑指南或隐藏技巧 | "套餐比单点划算很多，咸蛋黄南瓜建议避开，太油" |
| 5 分 | 含排队时间+最佳时段+点菜组合+停车信息等完整攻略 | "周末午市11点前到不用排队，3人点B套餐加一份辣子肥肠刚好，车停B2电梯直达" |

检查要点：
- [ ] 是否给出了具体的消费建议（是否值这个价）？
- [ ] 是否有避坑/隐藏菜单/最佳点菜组合？
- [ ] 是否有位置/交通/排队等实用信息？
- [ ] 是否提供了适合场景（聚餐/约会/一人食）？

### 维度 4: 真实性 (Authenticity)

衡量评论读起来是否像真人写的而非 AI 生成或情绪化审判。

| 分数 | 标准 | 示例 |
|------|------|------|
| 1 分 | 明显的 AI 风格或情绪化审判 | AI套话三件套，或"完全不值，最大的雷，网红店气质" |
| 2 分 | 大部分 AI 化或标签化贬低 | 全篇排比句+完美主义，或使用贬低标签 |
| 3 分 | 部分自然 | 有具体细节但结尾模板化，或知识融入生硬（"据了解..."） |
| 4 分 | 基本自然 | 语气真实但风格不够统一 |
| 5 分 | 完全像懂行的朋友在分享 | 温和有判断力、知识自然融入、批评就事论事、句子长短不一 |

检查要点：
- [ ] 是否有"非常好吃"、"环境优雅"、"服务周到"三联？
- [ ] 是否每道菜都好评（无批评即 AI）？
- [ ] 是否有"一定会再来"、"值得推荐"等模板结尾？
- [ ] 是否存在情绪化审判？
- [ ] 知识融入是否自然（"其实..."式 vs "据了解..."式）？
- [ ] 句子长度是否自然变化？
- [ ] 批评是否就事论事？

### 维度 5: 深度 (Depth) — v4.0 新增

衡量评论是否提供了普通食客注意不到的专业洞察。这是 v4.0 与 v3.0 的本质区别。

| 分数 | 标准 | 示例 |
|------|------|------|
| 1 分 | 纯语音转述，无任何专业补充 | "鱼肉嫩，酱油味重"（=用户原话） |
| 2 分 | 有1-2个形容词但无技术分析 | "鱼肉挺嫩，炒得不错" |
| 3 分 | 有1处技术观察或知识，但不深入 | "表面焦褐色，说明煎过"（观察到了但没解释为什么重要） |
| 4 分 | 多数菜品有图片证据+知识解释，专业增量≥30% | "焦褐色均匀说明煎焗火候控制精准——颜色不均通常是锅温不够或翻面时机不对" |
| 5 分 | 每道菜都有深度技术分析，专业增量≥50%，读者能学到东西 | "筷子夹起时鱼肉呈蒜瓣状分离——这是清蒸火候到位的标志。蒸过头肉就散了，蒸不够贴着骨头。这里控制得恰到好处" |

检查要点：
- [ ] 每道菜评价中，图片观察 + 知识解释的占比是否 ≥ 语音转述？
- [ ] 知识是否直接解释了图片证据或语音观点？（而非无关的"掉书袋"）
- [ ] 是否出现了普通食客注意不到的技术细节（焦化均匀度、酱汁分层、刀工一致性等）？
- [ ] 读者读完是否能学到至少一个专业判断标准？

**深度与杜撰的区分**：
- ✅ "表面焦褐色均匀"——图片可见证据
- ✅ "煎焗讲究锅温和时机"——料理公知
- ❌ "用了铸铁锅煎的"——图片无法判断锅具类型
- ❌ "主厨曾在XX餐厅工作"——无来源

## 杜撰检测 (Fabrication Check)

在四维度打分之前，必须先进行杜撰检测。将评论文本与结构化分析进行交叉验证：

**检测清单**：
- [ ] 评论中的每个具体价格 → 在图片（收据/菜单牌）或语音中有对应来源？
- [ ] 评论中的每个菜名 → 在语音（用户提到）或图片（可辨识）中有来源？
- [ ] 评论中的口味定论（"鲜嫩""酥脆""太咸"等）→ 是否来自用户语音原话或图片直接观察？
- [ ] 是否存在AI自行发挥的"脑补"描述（用户没说但AI推断的）→ 是否用"看起来""可能"标注了推测性质？

**杜撰判定**：
- 价格来源不明但写了具体数字 → **杜撰**，真实性直接扣到 2 分以下
- 菜名不在任何来源中 → **杜撰**，退回到生成阶段修正
- 用户说"还行"被写成"惊艳" → **偏离用户原意**，真实性扣 1 分
- 用户说"一般"/"不太好"被写成"最大的雷"/"完全不值" → **情绪夸大**，真实性扣 1 分
- 合理推测但标注了推测性质（"看起来像..."） → **不判定为杜撰**

## 综合评分规则

- **总分** = 具体性 + 生动性 + 实用性 + 真实性 + 深度（满分 25）
- **通过阈值**：总分 ≥ 18 且 每个单项 ≥ 3，且 深度 ≥ 3
- **低分触发器**：
  - 总分 < 18 → 全面优化
  - 深度 < 3 → 回到 Step 3 增加专业分析（这是 v4 的核心要求）
  - 其他单项 < 3 → 针对性修正该维度

## 评审报告格式

评审完成后，使用以下 JSON 格式输出评分报告：

```json
{
  "scores": {
    "specificity": <1-5>,
    "vividness": <1-5>,
    "usefulness": <1-5>,
    "authenticity": <1-5>,
    "depth": <1-5>
  },
  "total": <5-25>,
  "passed": <true/false>,
  "diagnosis": [
    {
      "dimension": "<维度名>",
      "score": <1-5>,
      "issue": "<具体问题>",
      "fix": "<修改建议>"
    }
  ]
}
```

仅对分数 < 3 的维度填写 diagnosis。diagnosis 可为空数组（全部 ≥ 3 时）。

**v4.0 特别关注**：深度 < 3 必须写 diagnosis。最常见的深度不足症状：
- 诊断："纯语音转述"，修改："回到 Step 3，为此菜补充图片技术证据 + 料理知识解释"
- 诊断："知识掉书袋"，修改："删除与图片/语音无关的知识，替换为解释具体观察的知识"
